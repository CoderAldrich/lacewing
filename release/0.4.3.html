Release Notes - 0.4.3

The 0.4.3 release of liblacewing is now available.

Windows support

0.4.3 has some important improvements for Windows support :-

  • Issues with the Windows implementation of FDStream have been resolved (solving things like truncation of Webserver file uploads)
  • There is now a Visual Studio project for each example, and all are contained in the master solution
  • Issues with building under MinGW have been resolved. Note that it is currently necessary to configure with --disable-spdy for a successful build.
  • Applications using the Server::Client::NPN function will now compile, although it will always return an empty string on Windows

Stream::Close behaviour

In previous versions of the library, the behaviour of Stream::Close was unclear when writes were still pending: some types of stream would close immediately and cancel the writes, while others would retain the internal structures until all pending writes had completed. This lead to confusion as to when, if writing to and then closing a stream, it would be safe to assume all pending data had been written.

To make this behaviour more controllable, Stream::Close now has a new boolean parameter immediately:

  • If true, the stream will cancel any pending writes. Stream::Close will always return true, and the stream is always closed.
  • If false, the stream will close as soon as all pending writes have completed. If no writes were pending, Stream::Close will return true and the stream is now closed. Otherwise, Stream::Close will return false, and the onClose handler can be used to observe out when the stream is finally closed.

Other changes

  • The public header file Lacewing.h is now lacewing.h (lowercase)
  • The C webserver examples have been updated for the current C API
  • Updated to the latest version of multipart-parser
