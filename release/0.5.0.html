Release Notes - 0.5.0

The 0.5.0 release of liblacewing is now available.

Reimplementation in C

Previous to 0.5.0, the library was written in C++98 with a set of wrappers to provide the C API. Starting with 0.5.0, this has been reversed: the library has been ported in its entirety to C99, with wrappers over the C functions to provide the C++ API.

API changes

Various parts of the API have been updated for consistency or correctness during the C port. Although a best effort has been made to keep the documentation up to date with these changes, there will be errors - so any reports of issues with the 0.5.x documentation are welcome.

One of the more visible API changes in 0.5.x is that all C++ classes and functions have been changed from CamelCase to snake_case to be consistent with the C API. The << stream operator is no longer supported, but a new printf-style writef function is provided instead.

C99 and MSVC

Microsoft Visual Studio does not support C99. In order to maintain Visual Studio support, liblacewing compiles as C++ under MSVC. To make this possible, any code targeting both UNIX and Windows is restricted to the subset of C99 that can compile as C++. In particular, the following C99 features are only used in the UNIX code:

  • Implicit coersion of void pointers (i.e. return value of malloc)
  • Designated initializers
  • Dynamically sized arrays

Other changes

  • The UNIX EventPump will now exit the event loop if epoll or kqueue throws an error
  • The various types of internal ref counting have been replaced with a more consistent lwp_retain and lwp_release
  • Preliminary support for SWIG
  • Updated SPDY and http-parser snapshots
  • write(3) is now used for non-sockets when SO_NOSIGPIPE is not available
  • Various fixes with webserver sessions
  • New ENABLE_SSL option in Makefile
