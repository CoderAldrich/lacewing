Using the Tag members

Nearly every class in liblacewing contains a member called Tag, and sometimes also internal.

The Tag member is initialized to 0 when an instance of a class is created, but is then left alone by the library. You are free to use it to store your own pointer (perhaps extending a Server::Client with your own client-specific state, for instance).

The internal member is used to hide implementation details from the application and keep public class sizes constant, and the library will use it to store a pointer to an internal structure. This should not be accessed by code outside of the library itself.

Most of the handlers that introduce new instances of classes (such as Server::onConnect) will state that they do so in the documentation, and when the last time the instance will be seen is (in this case, onDisconnect). This is a good time to free any data stored in Tag.

struct ClientState
{
    /* Your data here */
};

void onConnect (Lacewing::Server &, Lacewing::Server::Client &Client)
{
    Client.Tag = new ClientState;
}

void onDisconnect (Lacewing::Server &, Lacewing::Server::Client &Client)
{
    delete ((ClientState *) Client.Tag);
}

void onReceive (Lacewing::Server &, Lacewing::Server::Client &Client)
{
    ClientState &State = *(ClientState *) Client.Tag; /* Guaranteed to be valid */

}
