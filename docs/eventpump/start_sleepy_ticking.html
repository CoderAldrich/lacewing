start_sleepy_ticking (eventpump)

C++
lacewing::error lacewing::eventpump::start_sleepy_ticking
    (void (* on_tick_needed) (lacewing::eventpump))
C
lw_error lw_eventpump_start_sleepy_ticking
    (lw_eventpump, void (* on_tick_needed) (lw_eventpump *))

Description

Causes the eventpump to enter "sleepy ticking" mode with the specified callback.

Perhaps the most complicated method of using eventpump, this is suited to GUI applications (eg. Win32) which already have a message pump running in their main thread.

When sleepy ticking is enabled, the library will run a watcher thread to determine when a call to tick is necessary. The callback will then be called (from the watcher thread), and the application is then expected to call tick from the main application thread as soon as possible.

This could be accomplished, for example, in a Win32 application with PostMessage sending a custom window message in the callback, which then results in a call to tick when received by the main thread.

Error Handling - As eventpump has no hooks, any error will be returned directly from this function as a pointer to an error object. If no error occurred, 0 will be returned.

If an error does occur, it is the responsibility of the application to free the resulting error object using error_delete (C++) or lw_error_delete (C).

Parameters

callback

The callback, as described above.
