Lacewing::EventPump

Description

EventPump is the "heart" of the liblacewing library, providing the engine to drive all other classes.

Powered by IOCP, kqueue or epoll, it is in charge of things like notifying Server when data is available, and informing Timer when it should tick.

As there is no "one size fits all" model for doing this in all applications, EventPump provides a set of different methods for making it "live". These are :-

  • Tick - polls aggressively, suited to a game loop
  • StartEventLoop - starts a simple "block forever" event loop, suited to console applications and daemons
  • StartSleepyTicking - enables a threaded "please Tick later" callback, suited to GUI applications with an existing message pump

On Unix, it is also possible to opt out of all of these and use a custom method of watching file descriptors. A good example of this is the libev pump.
