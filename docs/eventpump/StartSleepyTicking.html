StartSleepyTicking (EventPump)

C++
Lacewing::Error * Lacewing::EventPump::StartSleepyTicking
    (void (LacewingHandler * onTickNeeded) (Lacewing::EventPump &))
C
lw_error * lw_eventpump_start_sleepy_ticking
    (lw_eventpump *, void (LacewingHandler * on_tick_needed) (lw_eventpump *))
JS
Function not currently supported

Description

Causes the EventPump to enter "sleepy ticking" mode with the specified callback.

Perhaps the most complicated method of using EventPump, this is suited to GUI applications (eg. Win32) which already have a message pump running in their main thread.

When sleepy ticking is enabled, Lacewing will run a "watcher thread" to determine when a call to Tick is necessary. The callback will then be called (from the watcher thread), and the application is then expected to call Tick from the main application thread as soon as possible.

This could be accomplished, for example, in a Win32 application with PostMessage sending a custom window message in the callback, which then results in a call to Tick on reception by the main thread.

Error Handling - As EventPump has no handlers, any error will be returned directly from this function as a pointer to an Error object. If no error occurred, 0 will be returned.

If an error does occur, it is the responsibility of the application to free the resulting Error object using delete (C++) or lw_error_delete (C). However, it is extremely rare for EventPump to throw an error, and most applications can probably ignore this case without consequence.

Parameters

Callback

The "tick needed" callback, as described above.
