onUploadPost (Webserver)

C++
void onUploadPost (Lacewing::Webserver &Webserver, Lacewing::Webserver::Request &Request,
                       Lacewing::Webserver::Upload * Uploads [], int UploadCount)
{
}

Webserver.onUploadPost (onUploadPost); /* to register the handler */
C
void on_upload_post (lw_ws * webserver, lw_stream * request,
    lw_ws_upload * uploads [], long upload_count)
{
}

lw_ws_onuploadpost (webserver, on_upload_post); /* to register the handler */

Description

The onUploadPost handler is triggered for an incoming HTTP POST request containing files.

This handler is the last time the Request object (as first seen in onUploadStart) will ever be passed to a handler. If EnableManualRequestFinish has been called, the Request object will be valid until either Request::Finish is called OR the onDisconnect handler triggers. Otherwise, the Request object is only valid until this handler returns.

The application may configure a response in this handler with functions such as Request::Header, and Request::Cookie, and write the response data with #Request::Write# and #Request::WriteFile#.

Nothing will actually be written to the socket while the Request object is still valid. This ensures that an accurate Content-Length header can be generated, and that headers can be changed throughout the lifetime of the Request.

The Uploads parameter is an array of Upload objects, all of which have finished uploading (either to the application through onUploadChunk and onUploadDone, or to a temporary file when Upload::SetAutoSave was called).
