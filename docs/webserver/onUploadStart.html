onUploadStart (Webserver)

C++
void onUploadStart (Lacewing::Webserver &Webserver, Lacewing::Webserver::Request &Request,
                         Lacewing::Webserver::Upload &Upload)
{
}

Webserver.onUploadStart (onUploadStart); /* to register the handler */
C
void on_upload_start (lw_ws * webserver, lw_stream * request, lw_ws_upload * upload)
{
}

lw_ws_onuploadstart (webserver, on_upload_start); /* to register the handler */

Description

The onUploadStart handler is triggered once for each file in an incoming HTTP POST request containing files.

When a POST request contains files, this handler is the first place the Request object will be seen by the application. The onUploadPost handler (which is where the response may be written) will be called only when all files have finished uploading.

This handler should not be used to write any response data, but it is the correct time to configure the Upload object (eg. calling Upload::SetAutoSave). If the application wishes to stop the upload, it may also (albeit drastically) sever the connection with Request::Disconnect.

If Upload::SetAutoSave is not called, this handler may be followed by onUploadChunk, and finally onUploadDone and (when any other uploads are also complete) onUploadPost itself. Otherwise, the Upload will next be seen in onUploadPost, when uploading to a temporary file is complete.
