on_upload_post (webserver)

C++
void on_upload_post (lacewing::webserver webserver, lacewing::webserver_request request,
                       lacewing::webserver_upload uploads [], int num_uploads)
{
}

webserver->on_upload_post (on_upload_post); /* to register the hook */
C
void on_upload_post (lw_ws webserver, lw_ws_request,
    lw_ws_upload uploads [], long num_uploads)
{
}

lw_ws_on_upload_post (webserver, on_upload_post); /* to register the hook */

Description

The on_upload_post hook is triggered for an incoming HTTP POST request containing files.

This hook is the last time the request object (as first seen in on_upload_start) will ever be passed to a hook. If #enable_manual_request_finish# has been called, the request object will be valid until either request::finish is called OR the on_disconnect hook triggers. Otherwise, the request object is only valid until this hook returns.

The application may configure a response in this hook with functions such as request::header, and request::cookie, and write the response data with #request::write# and #request::write_file#.

Nothing will actually be written to the socket while the request object is still valid. This ensures that an accurate Content-Length header can be generated, and that headers can be changed throughout the lifetime of the request.

The uploads parameter is an array of upload objects, all of which have finished uploading (either to the application through on_upload_chunk and on_upload_done, or to a temporary file when #upload::set_auto_save# was called).
