onDisconnect (Webserver)

C++
void onDisconnect (Lacewing::Webserver &Webserver, Lacewing::Webserver::Request &Request)
{
}

Webserver.onDisconnect (onDisconnect); /* to register the handler */
C
void on_disconnect (lw_ws * webserver, lw_ws_req * request)
{
}

lw_ws_ondisconnect (webserver, on_disconnect); /* to register the handler */
JS
webserver.onDisconnect (function (webserver, request)
{
});

Description

The onDisconnect handler is triggered when the connection that started a Request closes before the response has been sent (ie. #Finish# has been called).

onDisconnect is never called if manual request finishing has not been enabled, because then the lifetime of a Request object is not longer than the original handler (eg. onGet or onPost).

The exception to this rule is with multipart file upload, when onDisconnect may fire between the first onUploadStart and onUploadPost.

As onDisconnect only fires for an abnormally terminated request, it is a good way of detecting when a client using long-poll AJAX has left the page.
