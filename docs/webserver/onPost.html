onPost (Webserver)

C++
void onGet (Lacewing::Webserver &Webserver, Lacewing::Webserver::Request &Request)
{
}

Webserver.onPost (onPost); /* to register the handler */
C
void on_post (lw_ws * webserver, lw_stream * request)
{
}

lw_ws_onpost (webserver, on_post); /* to register the handler */

Description

The onPost handler is triggered for an incoming HTTP POST request.

This handler marks the beginning of the lifetime of a Request object. If EnableManualRequestFinish has been called, the Request object will be valid until either Request::Finish is called OR the onDisconnect handler triggers. Otherwise, the Request object is only valid until the handler returns.

The application may configure a response in this handler with functions such as Request::Header, and Request::Cookie, and write the response data itself with #Request::Send# and #Request::SendFile#.

Nothing will actually be written to the socket while the Request object is still valid. This ensures that an accurate Content-Length header can be generated, and that headers can be changed throughout the lifetime of the Request.
