Building liblacewing

Windows

There is a regular Visual C++ project file for MSVC 2008 or higher in the msvc directory. This has no external dependencies and should build out of the box, producing a DLL and LIB in the bin folder. Configurations are available for dynamic and static debug and release builds.

When linking statically, you should also add a blank preprocessor definition for lw_import to your project. This will prevent lacewing.h from adding __declspec(dllimport) to library functions.

Unix

Building on Unix-like operating systems (such as Linux, FreeBSD, OS X or even Windows/MinGW) should be as easy as:

./configure
make && make install
By default, the library will depend on OpenSSL. If SSL support is not required, this can be disabled with --disable-ssl: 
./configure --disable-ssl
To enable debugging symbols, configure with --enable-debug:

./configure --enable-debug

Android

It is possible to compile liblacewing for use with the Android NDK (not from Java). There is an Android.mk file provided in the jni folder, ready to use with ndk-build.

Unfortunately, the NDK is missing OpenSSL by default, so it is necessary to get hold of libssl.so, libcrypto.so and the openssl include folder before you can build.

A copy of these files is provided here for convenience. If this becomes unavailable (or if you'd rather get them yourself) you can get the shared objects easy enough from an Android device or emulator:

adb pull /system/lib/libssl.so
adb pull /system/lib/libcrypto.so

The openssl include directory can be obtained through git:

git clone -b donut http://android.git.kernel.org/platform/external/openssl.git

Install the files in [YOUR_NDK_ROOT]/platforms/android-*/arch-arm/usr (or somewhere else and add the search paths), and you should be good to go.
