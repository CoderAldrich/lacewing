What is the Lacewing Relay Protocol?

The Lacewing Relay Protocol is an experimental high-level networking protocol, with implementations in :-

  • C/C++ (as part of liblacewing)
  • ActionScript 3.0
  • Python

It was originally designed under the name "OINC" in 2008, sponsored by Clickteam for use with their multimedia development tools. Since then, it has been incorporated into the Lacewing project, and both the specification and implementations for various platforms have been made available.

The structure of a Lacewing Relay server is similar to an IRC network - each client has a name, and may join different channels. They may then send (TCP) or "blast" (UDP) messages, which the server will relay to their peers (others signed on to the same channel).

The protocol includes things like making peers aware of each other, framing messages, and features such as hidden channels and channel "masters".

The various implementations of the protocol are all interoperable, so eg. an ActionScript client could connect to a C++ server, and exchange messages with another client implemented in Python. Future plans for the protocol include the ability to run over WebSockets and/or HTTP (via long-polling).
