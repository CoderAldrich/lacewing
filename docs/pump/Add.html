    (int fd, void * tag, <read_callback>, <write_callback>, lw_bool edge_triggered)

Description

Adds a file descriptor to the pump. Applications should rarely need to use this directly.

Because of the nature of I/O completion ports vs. more traditional select-style event handling, the semantics are significantly different on Windows than every other platform. On Windows, there is a single callback, directly exposing the result of the underlying overlapped I/O:

    void on_completion (void * tag, OVERLAPPED *, unsigned int bytes_transferred, int error);

After registering a file descriptor, the user of the pump can issue operations such as ReadFile and WriteFile following standard overlapped I/O patterns, the callback being their completion routine.

On other platforms, two callbacks may be provided - one for read readiness and one for write readiness:

    void on_read_ready (void * tag);
    void on_write_ready (void * tag);

The user of the pump can then use regular system calls such as read(2) and write(2), with the confidence that they will probably succeed.

Parameters

FD

A standard file descriptor or, on Windows, a file or socket HANDLE.

tag

A pointer-sized tag to pass to the callback upon invocation.

edge_triggered [optional]

true for edge-triggered notifications (run the callback only when the FD first becomes ready), or false for level-triggered (run the callback continuously while the FD is still ready).

Generally speaking, with edge-triggered notifications one should repeatedly attempt to read or write until EAGAIN is returned, and then wait for the callback to fire before trying again. Internally, liblacewing nearly always uses edge-triggered I/O.

This option does not apply to Windows.
